<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR diff content → claude prompt | Untrusted diff content enters the model prompt; must use the SHARED fencing, not a per-adapter copy |
| `ANTHROPIC_API_KEY` → subprocess env / error text | Credential must reach the CLI via env and never appear on argv or in propagated stderr |
| Runner local context → claude session | Without `--bare`, `claude -p` auto-loads CLAUDE.md/MCP/skills from the runner, making review non-deterministic and potentially reading non-diff files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | Per-adapter prompt fencing drift | mitigate | ClaudeCodeAdapter calls shared `build_prompt` from `prompt.py`; acceptance criteria grep-asserts no local `_safe_diff_block`/`UNTRUSTED` definitions — single audited fencing copy (D-09). |
| T-05-06 | Information Disclosure | `ANTHROPIC_API_KEY` leak in error/argv | mitigate | Key passed via `env` dict only, never argv; `EngineFailure` messages routed through `sanitize_stderr(stderr, key)`; presence-only check raised as `ClaudeAuthError` (no key value in message). |
| T-05-07 | Tampering / Integrity | claude loads runner local context | mitigate | `--bare` is mandatory in the argv (Pitfall 3) — skips hooks/skills/MCP/CLAUDE.md autodiscovery and forces `ANTHROPIC_API_KEY` auth; argv acceptance criterion pins `--bare`. |
| T-05-08 | Tampering / DoS | unknown engine downgrade | mitigate | Registration goes through the same `get_adapter` fail-closed path (plan 01); unknown name still raises `UnknownEngineError`. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package install in this plan (adapter is pure stdlib; `uv.lock` unchanged). The `claude` CLI install (curl native installer) lands in plan 03's workflow YAML; Package Legitimacy Audit (RESEARCH) approved `curl https://claude.ai/install.sh | bash` (GPG-signed binaries). |
