<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR diff content → cursor prompt | Untrusted diff enters the prompt via a temp file; must use SHARED fencing |
| `CURSOR_API_KEY` → subprocess env / error text | Credential via env only, never argv or stderr |
| CLI install vector → runner | A malicious/impostor install vector would run third-party code with the API key in env |
| Cursor agent → runner filesystem | Without read-only posture the agent could write repo files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering / Supply chain | `cursor-agent` npm impostor package (`zalab-inc`) | mitigate (HIGH) | Install Cursor ONLY via `curl https://cursor.com/install -fsS \| bash`; workflow MUST NOT contain `npm install -g cursor-agent`; test_workflow_yaml.py statically asserts both (presence of curl installer, absence of npm install). Live task verifies `command -v cursor-agent` resolves the official binary (A1). |
| T-05-10 | Information Disclosure | `CURSOR_API_KEY` leak via argv/stderr/tempfile | mitigate | Key via `env` dict only; the temp file holds the PROMPT (already-fenced untrusted diff), never the secret; errors routed through `sanitize_stderr(stderr, key)`; temp file removed in `finally`. |
| T-05-11 | Elevation / Tampering | Cursor agent writes repo files | mitigate | `--force` omitted (read-only, A4); review is diff-in/findings-out with no PR-head checkout (SECR-01); live task confirms no file writes. |
