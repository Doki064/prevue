<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR diff content → engine prompt | Untrusted diff/file content crosses into the model prompt; must be fenced as UNTRUSTED DATA, never interpreted as instructions |
| `PREVUE_ENGINE` env value → adapter selection | A config string selects which credentialed CLI runs; a bad value must not silently downgrade the reviewer |
| Auth secret → subprocess env / error output | The credential must never appear on argv or in propagated error/stderr text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Hoisted `prompt.py` fencing (`_safe_diff_block`, `_escape_line`, build_prompt) | mitigate | Move VERBATIM (D-09); test_prompt.py + existing TestBuildPrompt assert build_prompt output byte-identical pre/post-hoist incl. UNTRUSTED DATA labels + contract-before-untrusted ordering. No per-module re-implementation. |
| T-05-02 | Tampering / DoS | `PREVUE_ENGINE` unknown value | mitigate | `get_adapter` raises `UnknownEngineError` naming the bad value + listing valid engines; no silent fallback to Copilot (D-04). test_registry.py `-k unknown` enforces. |
| T-05-03 | Information Disclosure | Auth secret in EngineFailure/stderr | mitigate | `sanitize_stderr(stderr, secret)` shared helper redacts the per-adapter secret; secrets passed via env dict, never argv (existing Copilot pattern preserved; test_copilot_adapter TestFailurePaths asserts token never echoed). |
| T-05-04 | Spoofing/Integrity | cli.py silently swallows non-Copilot auth errors | mitigate | cli.py catches shared `AuthError` base so any adapter's auth failure surfaces its specific message at exit 1 (Pitfall 5). |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (pure stdlib refactor; `uv.lock` unchanged). Cursor impostor supply-chain threat is modeled in plan 03 where the install step lands. |
</threat_model>
