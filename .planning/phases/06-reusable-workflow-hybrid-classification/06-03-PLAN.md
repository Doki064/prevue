<objective>
Deliver the hybrid classification vertical slice (CLSF-02): a per-file LLM
fallback that fires only for files no glob rule matched, routed through the
already-selected engine adapter with a cheap model, degrading to `general` +
disclosure on any failure. This plan also repoints `run_review` to the
single-read config loader from 06-01, closing the `review.py:45-47` consumer-path
gap (WKFL-03 wiring) so the fallback and config land together as one working
end-to-end path.

After this plan: a PR with an ambiguous file (no glob match) gets a cheap
classification call for just that path; a PR whose files all match stays
zero-token; and any classification failure leaves the review running with a
`general` label and an explicit "classification fallback unavailable" note —
never a red check (D-12, distinct from the fail-closed review path).

Purpose: This is the core thesis enhancement — "ambiguous diffs fall back to a
cheap LLM call; clear-cut PRs spend zero classification tokens" (CLSF-02). It
reuses the Phase 5 adapter (D-10) and the new `classify()` capability (D-11) from
06-01 — no new dependencies.

Output: per-adapter `classify()` implementations, new `classify/llm_fallback.py`,
`run_review` wired to single-read config + fallback + disclosure, and green
`tests/test_llm_fallback.py` + extended contract/review-flow suites.

Executor note (scope awareness): this plan touches 9 files across 4 adapters, a
new module, and the review.py wiring; Task 1 (classify() on 3 adapters + the
contract-suite extension) is the densest unit because the three adapter
implementations share one canonical-label validation path and one parametrized
mock-subprocess fixture — kept together deliberately so the validation/fixture
logic is written once, not triplicated. If execution strains context, the clean
seam is to land the three adapter `classify()` methods first, run ruff, then add
the parametrized contract cases in a second pass within the same task — do NOT
spill adapter work into a separate plan, which would duplicate the shared fixture.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR file paths → cheap classification model | Unmatched file paths cross into an LLM CLI prompt |
| engine CLI stderr → logs | Subprocess stderr may contain the auth token |
| cheap-model output → routing | Free-text LLM labels must be constrained before routing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Tampering | classify prompt (untrusted paths) | mitigate | Reuse build_classify_prompt fencing (`~~~UNTRUSTED DATA` + `_escape_line`); paths never treated as instructions (06-01 contract) |
| T-06-10 | Information Disclosure | classify() subprocess stderr | mitigate | Reuse sanitize_stderr(stderr, token) on nonzero exit (same as review(); errors.py) |
| T-06-11 | Spoofing/Tampering | non-canonical LLM labels | mitigate | Validate every returned label ∈ CANONICAL_LABEL_ORDER, drop unknowns (Pitfall 6); single source of truth for allowed labels |
| T-06-12 | Denial of Service | fallback failure blocking the PR | mitigate | Best-effort degrade to `general` + disclosure on any failure (D-12); classification exception NEVER propagates to the gate; distinct from fail-closed review path |
| T-06-13 | Elevation of Privilege | config read location | mitigate | PREVUE_CONFIG_PATH resolves under the consumer base-ref checkout (set absolute by the workflow); cwd/PR-head config is not read (D-04/D-07) |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new package installs (RESEARCH audit: zero new deps; stdlib subprocess + installed adapters only) |
