<objective>
Deliver the shippable surface: the `on: workflow_call` reusable workflow plus the
thin `pull_request` caller that dogfoods it on this repo. This is a complete
vertical slice — after this plan a real (consumer) repo can adopt Prevue with a
minimal caller snippet pinned to a release tag and get the full review pipeline
under minimal permissions.

The reusable workflow self-checks-out the Prevue Python code at its matching
release tag (D-03 version-skew guard) and the consumer repo at
`pull_request.base.sha` (D-04, never PR head), maps the chosen engine's named
secret to its native env var (D-02, no `secrets: inherit`), and runs a single
`uv run prevue review`. Draft PRs skip at the workflow `if:` (D-13, free).

Purpose: This is the first externally shippable interface (WKFL-01/02/04) and the
delivery mechanism the whole project is built to provide. Dogfooding via the thin
caller proves consumers hit the exact path this repo runs.

Output: `.github/workflows/prevue-review.yml` (new), `.github/workflows/review.yml`
(rewritten as thin caller), `docs/consumer-setup.md` (new), green
`tests/test_reusable_workflow_yaml.py`, extended `tests/test_workflow_yaml.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| consumer caller → reusable workflow (`workflow_call`) | Consumer passes inputs + named secrets across the workflow boundary; the reusable workflow declares its own least-privilege permissions |
| consumer repo PR head → checkout | The base ref (trusted) is checked out for config; PR-head code must never be checked out for analysis |
| named secrets → engine CLI env | Per-engine credentials flow only to the selected engine's native env var |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Elevation of Privilege | consumer checkout ref | mitigate | Checkout consumer at `pull_request.base.sha` only, persist-credentials:false; PR-head code never checked out (D-04/SECR-01); static test asserts no PR-head checkout |
| T-06-05 | Information Disclosure | secrets pass-through | mitigate | Named per-engine secrets `required:false`; NO `secrets: inherit` (CLAUDE.md); static test asserts absence of `secrets: inherit` |
| T-06-06 | Elevation of Privilege | trigger model | mitigate | `pull_request` trigger only in the caller; NO `pull_request_target`; static test asserts absence (SECR-01) |
| T-06-07 | Tampering | version skew (self-checkout ref) | mitigate | Hardcode Prevue self-checkout ref to a pinned release tag (not main/HEAD); static test asserts ref is a pinned tag; `prevue-ref` override is test-only (D-03, Pitfall 1) |
| T-06-08 | Tampering | supply-chain (actions pins) | mitigate | Every `uses:` SHA-pinned to the same SHAs as review.yml; zizmor + actionlint CI gates cover the new file (Pitfall 7) |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new package installs introduced by this phase (RESEARCH audit: zero new deps); the in-workflow `npm install -g @github/copilot@1.0.61` and curl installers are reused verbatim from the already-audited review.yml |
</threat_model>
