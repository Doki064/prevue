## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| consumer skill files → review prompt | Consumer skill content is TRUSTED instructions (consumer owns the base-ref repo, D-11) — but ONLY when read from the base ref, never PR head |
| PR head skill edits → run | A skill file modified on the PR head must NEVER be executed in the same run (SKIL-04) |
| oversized consumer skill → prompt budget | A hostile-large consumer skill must not crowd out the diff/output budget |

| T-07-08 | Tampering | consumer skill source path | mitigate | Consumer skills loaded from the base-ref checkout (PREVUE_CONSUMER_ROOT), never PR head (SKIL-04/D-01) — a PR cannot inject a skill into its own review. Document in SECURITY.md (Plan 05) |
| T-07-09 | Denial of Service | oversized consumer skill | mitigate | Per-skill + total + count caps with skip-and-disclose (D-07); enforced in the loader, defaults set in Plan 01 |
