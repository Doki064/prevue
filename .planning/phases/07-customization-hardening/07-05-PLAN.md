## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client/PR → engine prompt | PR diff hunks, filenames/paths, and the classify fallback prompt are UNTRUSTED; only top-of-prompt instructions are authoritative (D-11) |
| engine CLI → PR metadata/network | The engine's default headless tool posture must not reach PR title/body/comments (excluded from DiffBundle) or the network (vector 4) |
| engine findings → verdict | Findings are untrusted output; the gate verdict is computed independently and findings can only reference real changed lines |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-injection-1 | Tampering / Elevation | diff hunk content (vector 1) | mitigate | 4-backtick diff fence + backtick-run escaping (_safe_diff_block) + INSTRUCTION_REASSERTION tail (D-10); regression-guarded in test_injection_adversarial |
| T-07-injection-2 | Tampering | filenames/paths (vector 2) | mitigate | json.dumps escaping (_escape_line) + paths fenced in UNTRUSTED block + reassertion tail |
| T-07-injection-3 | Tampering | classify fallback prompt (vector 3) | mitigate | build_classify_prompt already fences paths (verified) + reassertion tail added |
| T-07-injection-4 | Information Disclosure / Elevation | engine tool reach to PR metadata/network (vector 4) | mitigate | Live human-verify of each adapter's default headless tool posture (Task 2, A1); explicit deny flag added if any tool reachable; recorded in SECURITY.md |
| T-07-injection-5 | Tampering (false pass) | malicious finding on unchanged line | mitigate | positions.build_valid_lines drops invalid positions to position-fallback; gate verdict computed independently (regression-guarded) |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new package installs (07-RESEARCH Package Legitimacy Audit: N/A — all deps pinned). No legitimacy checkpoint required. |
