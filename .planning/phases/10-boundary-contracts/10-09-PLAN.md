<objective>
Fix the one remaining `human_needed` gap from `10-VERIFICATION.md`: the Copilot OTEL usage parser (`usage.py::_parse_copilot_otel`) parses a schema the real Copilot CLI (`gh copilot`, verified locally against v1.0.67) never produces. The real file exporter writes FLAT per-line `{"type": "span"|"metric", ...}` records with `attributes` as a plain dict (not a list of `{key,value}` pairs) and real token keys `gen_ai.usage.input_tokens` / `.output_tokens` / `.cache_read.input_tokens` / `.cache_creation.input_tokens` (span attrs also carry `github.copilot.cost` as a USD float). Because real records never contain `resourceSpans`, `span_count` stays 0 and the parser always returns `None`, silently degrading to the bytes/4 estimate regardless of whether `COPILOT_GITHUB_TOKEN`/`COPILOT_OTEL_FILE_EXPORTER_PATH` are correctly wired.

This is a **code-only fix** — the prior 10-08 gap-closure plan flipped `copilot-cli`'s `usage_capture` to `"none"` based on a since-superseded conclusion ("OTEL has no effect on 1.0.61"); that conclusion is now known to be stale — OTEL export is real and documented (`copilot help monitoring`) as of at least 1.0.67. This plan reverses that flip and fixes the actual parser bug underneath it.

Purpose: Deliver real (non-estimated) Copilot token/cost accounting per PERF-03/D-04, which the phase already declared as its primary path ("estimation is NOT the primary path... `bytes/4` is a labeled fallback").
Output: A rewritten `_parse_copilot_otel` matching the real flat-span JSONL schema, `copilot-cli`'s spec flipped back to `usage_capture="otel-jsonl"`, `COPILOT_OTEL_FILE_EXPORTER_PATH` wired into both reusable workflow YAML files with a per-invocation temp path, updated test fixtures/tests, and updated docs — with the live-CI spot-check explicitly recorded as still open.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Copilot CLI subprocess (untrusted external binary) -> OTEL JSONL file on disk -> `_parse_copilot_otel` | The parser consumes output written by an external CLI process. Malformed, adversarial, or unexpectedly-shaped JSONL lines must never crash the review or misattribute token/cost data. |
| `COPILOT_OTEL_FILE_EXPORTER_PATH` env value (workflow-YAML-controlled, not PR-controlled) | The path is a `runner.temp`-derived directory set by the trusted reusable workflow YAML, never sourced from PR content or consumer config — no injection surface from a PR author. |
| Workflow YAML env-block edits (`.github/workflows/prevue-review.yml`, `prevue-command-run.yml`) | Adding an unconditional env var to a step must not weaken existing per-engine secret gating (`COPILOT_GITHUB_TOKEN` etc. remain conditional on `inputs.engine == 'copilot-cli'`); only the inert OTEL path is set unconditionally. |

No single malformed line can raise an uncaught exception (T-10-07 precedent preserved, extended to the new flat schema).
