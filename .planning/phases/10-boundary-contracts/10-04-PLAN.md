<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR-head prevue.yml → review config | A malicious PR may edit prevue.yml to inject CLI flags (raw_args) or fake pricing. Must be read only from the trusted base ref. |
| engine.raw_args list → CLI subprocess argv | Caller-controlled (base-ref) flags appended to the engine command. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-09 | Tampering / Elevation | PR-head config injects CLI flags via raw_args | mitigate | raw_args parsed from the single `load_config` read which is gated by base-ref-only `resolve_consumer_config_path` (sentinel under Actions w/o PREVUE_CONSUMER_ROOT). Test asserts PR-head raw_args is ignored (Pitfall 4). |
| T-10-10 | Tampering | command injection through raw_args | mitigate | List form only (D-10) — appended to the argv list; no shell string, no `shell=True` (invoke_subprocess_text uses a list). Validator rejects a string raw_args. |
| T-10-11 | Tampering | PR-head config fakes pricing to hide cost | mitigate | `engine.pricing` parsed from the same base-ref-only gated read; `ConfigDict(extra="forbid")` validates the map; no runtime fetch. Test asserts PR-head pricing is ignored. |
| T-10-12 | Information Disclosure | model-tiering leaks a wrong/expensive model silently | accept | Per-role model is a config value resolved with declared precedence; mis-set model is a config error surfaced in the sticky model label, not a security issue. |
| T-10-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (per 10-RESEARCH Package Legitimacy Audit). |
</threat_model>
