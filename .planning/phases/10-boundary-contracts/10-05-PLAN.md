<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| review output → $GITHUB_OUTPUT / artifact | Prevue-controlled serialized result consumed by downstream automation; must not enable output injection or overflow. |
| consumer automation → job outputs | Consumers chain `if:` on the compact output; it must be a stable versioned contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-13 | Tampering | $GITHUB_OUTPUT injection via newline in a value | mitigate | Compact values are scalars; any multiline value uses the GitHub `name<<DELIM` heredoc form with a random delimiter; `emit_machine_output` never writes a raw multiline scalar (test asserts newline-safe lines). |
| T-10-14 | Denial of Service | job-output overflow on large reviews | mitigate | Only the compact form (counts + totals) goes to `$GITHUB_OUTPUT`; the full findings JSON goes to an artifact (D-08, 1 MB limit, Pitfall 6). |
| T-10-15 | Information Disclosure | artifact leaks secrets in engine_meta | mitigate | `build_full_output` serializes `ReviewResult` (summary/findings/engine_meta numeric token+cost only); no secret material is in engine_meta (sanitized upstream). The artifact contains the same data already posted to the PR. |
| T-10-16 | Tampering | versionless contract silently breaks consumers | mitigate | `schema_version="1.0"` on both forms (D-09) so consumers can gate on the version; documented field shape. |
| T-10-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages. `actions/upload-artifact` is a SHA-pinned GitHub action (matches existing pin convention); not a package install. |
</threat_model>
