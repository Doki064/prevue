<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| issue_comment event → workflow execution | Runs in trusted default-branch context; must never check out PR head (Fact 1) |
| comment body → workflow steps | Passed as env data only; never interpolated into a shell `run:` (Fact 4) |
| GITHUB_TOKEN scopes → command actions | Minimal scopes only; no contents: write, no pull_request_target (Fact 6) |
| commenter identity → privileged run | author_association pre-filter in `if:`; authoritative permission gate in Python |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-08-40 | EoP | unauthorized trigger of a privileged review/dismiss/resolve | mitigate | `if:` author_association pre-filter + authoritative collaborator-permission gate in Python (08-14/08-15); §L7 checkpoint proves a read-only collaborator is denied. HIGH per security_block_on — BLOCKING human checkpoint. |
| T-08-41 | EoP | fork-PR command running code in base context | mitigate | No PR-head checkout step exists (Fact 1); run_command refuses fork PRs (SECR-01); §L7 step 3 verifies no head checkout. HIGH. |
| T-08-42 | Tampering/EoP | shell/template injection via comment body (Fact 4 / Pitfall 7) | mitigate | Body only ever reaches Python via PREVUE_COMMENT_BODY env; zizmor template-injection gate in CI; never referenced in a `run:` line. HIGH. |
