<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| commenter → privileged dispatch | Every command path runs behind the write-access gate, executed first |
| comment body → review/dismiss/resolve action | Parsed in Python; ids validated; no shell |
| fork PR → base-context execution | Refused before any work (SECR-01) |
| dismiss creation → suppress-list write | Only an existing finding can be dismissed; 🔴 needs a reason; bounded |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-35 | EoP | unauthorized user runs review/dismiss/resolve | mitigate | authorize_commenter (Fact 5) runs FIRST in run_command; unauthorized → posted reply + exit 0, zero engine/dismiss/resolve. HIGH per security_block_on — live-verified in 08-16 §L7. |
| T-08-36 | Tampering | dismissing a finding that does not exist (place real findings go to die) | mitigate | Guard 1: create_dismiss_entry refuses an ident with no live finding/thread this run; 🔴 requires an explicit reason; audited in the sticky. |
| T-08-37 | EoP | fork PR command executing in base context | mitigate | run_command refuses when head_repo_full != base_repo_full before any engine/checkout work (SECR-01). |
| T-08-38 | Tampering | argument injection of <id> into resolve/engine | mitigate | parse_command already validates ids ([0-9a-f]{16} / [A-Za-z0-9_=-]{8,}, 08-14); create_dismiss_entry/resolve only accept a validated ident matched against re-derived priors. |
