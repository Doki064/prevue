<objective>
Build the LIFE-01 state + scope-decision layer (marker SHA read/write, the
full-vs-incremental ancestry classifier), the D-11 gate-over-open-set, and the
three new config knobs.

Purpose: These are the deterministic decision units (TDD candidates per
VALIDATION.md): the marker is the ONLY persisted state (SHA only, D-01);
decide_scope is the ancestry classification that prevents reviewing a bogus
incremental range after a force-push (D-03); the gate-over-open-set closes the
headline false-green hole (D-11). Config knobs land here because Plans 04/05/06
consume them. Depends on 08-01 for the compare fixtures.
Output: parse_marker_sha/render_marker (comments.py), decide_scope + incremental
file-set path (diff.py), open-set gate input (gate.py), config knobs (gate.py
ReviewConfig + config.py loader).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sticky marker body → last_sha | The persisted SHA is parsed from the sticky comment and used in a compare API call |
| compare API response → in-scope file set | GitHub-derived status/merge_base drive the full-vs-incremental branch |
| carried prior findings → gate verdict | Severities re-derived from untrusted live comments feed the pass/fail decision |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | marker SHA injection into the compare URL path | mitigate | _MARKER_RE only matches [0-9a-f]{7,40}; parse_marker_sha returns None on anything else, so a non-hex value never reaches repo.compare (ASVS V5). |
| T-08-06 | Tampering | force-push producing a bogus lastSHA..head range | mitigate | decide_scope requires status=="ahead" AND merge_base_commit.sha==last_sha for incremental; every other case falls back to full + marker reset (D-03). Failing safe to full is never wrong. |
| T-08-07 (gate false-green) | Tampering/EoP | clean incremental push over an unresolved prior error | mitigate | apply_gate is fed union(new, carried-unresolved) − resolved-this-run; conclude ladder keeps the check non-green while an unresolved error stands (D-11, extends P7 D-23).
