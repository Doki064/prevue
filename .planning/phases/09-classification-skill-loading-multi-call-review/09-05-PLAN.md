## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR diff → N per-call prompts | Untrusted diff fans out into multiple engine subprocess calls |
| consumer caps → concurrency/call count | Consumer-controlled review_concurrency/max_review_calls drive subprocess fan-out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-15 | Denial of Service / cost amplification | huge PR → many parallel subprocesses / token spend | mitigate | review_concurrency cap (ThreadPoolExecutor max_workers) + max_review_calls cap + max_total_run_tokens whole-run ceiling (D-09/D-10); overflow drops lowest-priority files with disclosure rather than spawning unbounded calls |
| T-09-16 | Tampering (prompt injection) | each of N per-call prompts | mitigate | Every per-call ReviewRequest routes through build_prompt UNTRUSTED DATA fencing + INSTRUCTION_REASSERTION (prompt.py) — no raw concatenation; per-call instructions ride on the request, no global mutation |
| T-09-17 | Repudiation / false-green | a failed call silently dropping findings | mitigate | Fail-soft marks the run degraded → neutral when failures>0 (D-08); merge severity tie-break (Pitfall 4) prevents an error being masked by a duplicate warning |
| T-09-18 | Information Disclosure | secret leakage in multi-call stderr | mitigate | Each call uses the same adapter invoke path; existing sanitize_stderr / token redaction in EngineFailure unchanged |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (concurrent.futures stdlib); `uv sync --locked` |
