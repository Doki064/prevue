<objective>
Implement the B+D hybrid skill selection module (D-02): a deterministic zero-token keyword floor over skill `name`+`description`+`applies-to`, with LLM escalation for below-threshold skills that belong to a routed bundle (the gap-closure guard). Mirrors the proven CLSF-01 (deterministic) + CLSF-02 (LLM fallback) classification architecture.

Purpose: This is the Thread 1 core algorithm that closes the SKIL-01 gap — a routed bundle's relevant-but-low-keyword-score skill (the gap-demo-sandbox shape) gets a chance to load via escalation instead of being silently dropped by glob-only matching. Whole-run token optimization: the keyword floor avoids loading whole bundles of irrelevant skill bodies into the review prompt.
Output: `src/prevue/skills/selection.py` (pure functions, TDD), and an extension to `llm_classify` (or a new `llm_select_skills`) so the already-firing classify fallback also returns relevant skill names (double-duty).
</objective>
