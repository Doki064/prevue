<objective>
Implement the lightweight, safe import-scan module (D-06/ENGN-06): extract import/reference targets from a changed file's patch so the multi-call splitter (Plan 09-05) can co-locate files that directly reference each other in the same review call, preserving cross-file context.

Purpose: ENGN-06 requires context-preserving splitting — files that share imports/references must land in the same call. This module is the cheap static analysis (stdlib `ast` for Python, tolerant regex for JS/TS and others) that feeds the splitter. It MUST degrade gracefully (no parser → no references for that file → bundle-only grouping) and MUST be safe against attacker-controlled diff fragments (parse, never execute).
Output: `src/prevue/importscan.py` with `extract_imports()` and `referenced_paths()`, TDD with explicit degrade-on-garbage and no-exec safety tests.
</objective>

  <behavior>
    extract_imports(path, patch):
    - Test (Python ast): a .py patch with added `import foo.bar` and `from a.b import c` yields module strings ["foo.bar", "a.b"] (or equivalent normalized). Prefer ast.parse on the reconstructed added-line text when it parses; wrap in try/except SyntaxError.
    - Test (Python regex fallback): a .py patch whose added lines do NOT form a parseable module (partial hunk) falls back to a tolerant regex over `^\+\s*(import|from)\s+...` added lines and still extracts the module names — does NOT raise.
    - Test (JS/TS regex): a .ts/.tsx/.js/.jsx patch with added `import X from "./util"` and `const y = require("../lib/db")` yields specifiers ["./util", "../lib/db"].
    - Test (unknown language): a .rs/.go/.unknown patch returns [] (no parser → empty), no crash.
    - Test (patch is None): returns [] (binary/omitted patch).
    - Test (garbage/adversarial): a patch with embedded backticks, fake `__import__`, os.system text, or a giant line returns extracted specifiers or [] but NEVER executes anything and NEVER raises.
    referenced_paths(files):
    - Test: given ChangedFiles a.py (imports module b) and b.py present in the set, the result maps a.py → {b.py} (resolve import module/relative specifier to a changed-file path when one matches; otherwise omit).
    - Test: a reference to a module NOT in the changed-file set is ignored (only co-locate files that are BOTH in the diff — D-06 cross-file locality within the call set).
    - Test: relative JS specifiers (./util, ../lib/db) resolve against the importing file's directory to a changed-file path when present.
    - Test: the relation is used directionally but co-location is symmetric for grouping (document which the splitter consumes).
  </behavior>

Wrap the whole per-file scan in try/except so one bad file degrades to no-refs, not a run failure.
