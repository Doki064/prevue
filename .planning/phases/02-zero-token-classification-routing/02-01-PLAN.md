## Phase Goal

**As a** developer reviewing a clearly-typed PR, **I want to** have it automatically classified into category labels and routed to skill bundles deterministically at zero token cost, **so that** the review loads only the guidance the PR actually needs and I can see exactly which labels and rules were applied.

## Acceptance Criteria (this slice)

- [ ] `pathspec==1.1.*` and `PyYAML==6.0.*` are declared in `pyproject.toml` and installed (DIFF-02/CLSF-01 prerequisites).
- [ ] A new `src/prevue/classify/` package exists with `RuleSet`/`ClassificationResult` models, a `default_rules.yml` data file, rule loading, and thin filter/classify/route pure functions.
- [ ] `run_review()` threads a real diff through filter→classify→route and the resulting labels + matched globs render in the sticky-comment Metadata section (D-09).
- [ ] The end-to-end `run_review` test proves the engine receives a *reduced* (filtered) DiffBundle (D-08) and the Metadata shows the assigned label(s).
- [ ] Zero subprocess/network calls occur inside the classify stage — zero-token is structural, not measured.

<objective>
Establish the deterministic classification stage end-to-end as the thinnest working vertical slice: add the two new dependencies, ship the built-in rules as a YAML data file, model the ruleset/result with pydantic, build thin filter/classify/route pure functions, and wire the whole filter→classify→route→render path into `run_review()` so a real PR's labels and matched rules surface in the sticky comment's Metadata section.

This plan deliberately ships *thin* implementations (first-match single-pass classify, 1:1 routing, no consumer-config merge yet) so the full path works end-to-end. Plan 02 refines classification (multi-label union D-01, PR-level `general` D-03, provenance ordering) and Plan 03 refines filtering/edges (D-08 hardening, D-10 empty-PR skip, consumer additive merge D-05/D-07, routing precedence D-06).

Purpose: Prove the zero-token classify→route→audit thread works against a real `DiffBundle` through `run_review` before refining edge behaviors. MVP vertical-slice discipline — a working end-to-end path first, refinements after.
Output: `pyproject.toml` (deps), the new `src/prevue/classify/` package (6 files incl. `default_rules.yml`), modified `review.py` + `comments.py`, and extended/new tests.
</objective>
