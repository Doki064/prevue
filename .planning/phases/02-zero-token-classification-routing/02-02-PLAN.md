## Phase Goal

**As a** developer reviewing a multi-domain or unclassifiable PR, **I want to** receive every applicable category label (or a deterministic `general` fallback when nothing matches), **so that** no domain is under-reviewed and no PR is left without a route, all surfaced in a stable, auditable order.

## Acceptance Criteria (this slice)

- [ ] `classify` returns the UNION of all matched labels for a multi-domain PR (D-01) — no single-dominant tie-break.
- [ ] `classify` returns `{general}` only when the whole PR matched zero real rules (D-02/D-03), never alongside real labels.
- [ ] Labels are emitted in a fixed canonical order so the sticky comment does not churn across runs (RESEARCH Pitfall 5).
- [ ] Matched-glob provenance is preserved per label for the D-09 audit trail.
- [ ] `route` maps `general` → the `general` bundle and preserves deterministic ordering.

Update the router to handle the `general` label and update the Metadata renderer to use the canonical order.

This is a refinement slice over Plan 01's working end-to-end path — the same path now produces correct multi-domain and fallback results.

Purpose: D-01/D-03 are the heart of "load only what's needed" without under-reviewing a secondary domain, and the `general` label is the deliberate, named hand-off seam to Phase 5's LLM fallback. Getting union-vs-general right (RESEARCH Pitfall 3) is the highest-value correctness work in the phase.
