<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Finding text → consumer-visible markdown | Engine-authored (untrusted-influenced) titles/bodies/suggestions enter the sticky table, details blocks, and inline comments |
| GateResult → GitHub Reviews API | Rendered payloads cross into authenticated writes on the PR |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering | findings table cells | mitigate | `_escape_table_cell` escapes pipes and strips newlines — engine text cannot add rows/columns or break out of the table (RESEARCH Security Domain markdown-injection row) |
| T-04-12 | Tampering | suggestion fences | mitigate | `_safe_suggestion_block` 4-backtick fence with ``` escaping (reuses the proven `_safe_diff_block` pattern) — suggestion text cannot close the fence and inject instructions/markup |
| T-04-13 | Denial of Service | comment body size | accept | GitHub hard-caps comment bodies at 65,536 chars and rejects oversized payloads server-side; budget cap (≤10 inline default) + table one-liners bound realistic sizes; explicit length capping deferred unless live verification shows otherwise |
| T-04-14 | Information Disclosure | create_review failure handling | mitigate | Warning logs failure class + comment count only (payload shape, never finding content); no PyGithub exception text re-raised unsanitized |
| T-04-15 | Spoofing | verdict drift | mitigate | Verdict strings imported from gate.py (D-07) — the sticky cannot show a different verdict than the check run concludes |
</threat_model>
