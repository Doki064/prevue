    - adapter: hard failure on the FIRST invocation (timeout/nonzero/empty) → EngineFailure raises as today (D-04 distinct classes)
    - adapter: hard failure on the RETRY invocation → degrade with attempt-1 prose, do not raise (best-effort retry; "never a crash")

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine stdout → parser | LLM output influenced by untrusted diff content crosses into typed models |
| diff content → engine prompt | Untrusted PR data could attempt to inject decoy fences or attacker findings |
| parse errors → retry prompt | Error feedback re-enters the engine prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02 | Tampering | extract_json_fence | mitigate | Last-fence extraction + fence-at-end instruction (Pitfall 5) defeats decoy fences echoed in prose; non-list top level rejected |
| T-04-03 | Tampering/Spoofing | prompt-injected findings | mitigate | Untrusted-data fencing preserved byte-identical; D-15 rubric is a module constant, never consumer-interpolated; strict Literal severity bounds gate impact; path/line membership validated downstream (04-03); budget cap bounds volume (04-03) |
| T-04-04 | Information Disclosure | retry feedback + new error strings | mitigate | Feedback appendix contains only the parser's own error text (json module / pydantic messages), never raw stderr; any subprocess-derived text passes `_sanitize_stderr` token redaction |
| T-04-05 | Denial of Service | retry loop | mitigate | Exactly one retry (D-02, 2x spend ceiling); MAX_PROMPT_BYTES re-checked before retry; salvage never retries |
| T-04-06 | Repudiation | fully-dropped salvage | mitigate | 0-valid-of-N>0 → degraded=True (neutral check downstream), never a clean empty result (false-green avoidance, RESEARCH Open Q1) |
</threat_model>
