<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| consumer prevue.yml → ReviewConfig | Consumer-authored YAML crosses into gate policy |
| engine findings → placement decisions | Engine-emitted (untrusted-influenced) paths/lines decide where GitHub writes land |
| GateResult → branch protection | The conclusion directly controls merge-gate behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Elevation of Privilege | ReviewConfig / load_review_config | mitigate | Fail-closed pydantic at startup before engine spend: extra="forbid", Literal severity, ge=0 (D-16); trusted-base-ref read posture unchanged (ASVS V5) |
| T-04-08 | Tampering | apply_gate position validation | mitigate | Path-membership + per-side line-set checks against real unidiff hunks (D-17) — hallucinated/injected paths and lines can never become inline comments and would otherwise 422 the whole atomic review POST (Pitfall 2) |
| T-04-09 | Denial of Service | inline volume | mitigate | Hard budget cap (D-18 default 10, ge=0) bounds the blast radius of injected findings; deterministic D-19 allocation, no unbounded loops |
| T-04-10 | Repudiation/Tampering | conclude ladder | mitigate | Verdict computed over ALL validated findings (D-14); degraded always neutral, never success (false-green avoidance); only explicit opt-in `min_severity_to_fail` can block (D-05/D-06) — no accidental red X |
</threat_model>
