## Phase Goal

**As a** developer who opens a pull request, **I want to** automatically receive an AI-generated review summary comment, **so that** I get feedback without anyone running a review manually.

## Acceptance Criteria (this slice)
- [ ] The diff + changed-file metadata is fetched via the GitHub API as data (no untrusted checkout) â†’ `DiffBundle` (DIFF-01).
- [ ] One sticky comment is posted and updated in place across runs via a hidden marker (OUTP-01, D-06).
- [ ] The sticky body is a sectioned shell (Verdict/Review/Metadata) with "no verdict in v1" (D-04/D-05).

<objective>
Build the two deterministic GitHub I/O units that bracket the engine call: reading the PR diff as API data (no checkout) into a `DiffBundle`, and writing exactly one sticky summary comment via a hidden-marker upsert. Both are pure-boundary, mockable units â†’ test-first.

Purpose: These are the "fetch" and "post" ends of the walking-skeleton loop; deterministic Python owns all GitHub writes (P3) so the engine never touches GitHub.
Output: `github/client.py`, `github/diff.py`, `github/comments.py` + their unit tests using `responses`/mock PR.
</objective>
