## Phase Goal

**As a** developer who opens a pull request, **I want to** automatically receive an AI-generated review summary comment, **so that** I get feedback without anyone running a review manually.

## Acceptance Criteria (this slice)
- [ ] The workflow uses `pull_request` only, minimal permissions, pinned SHAs, and runs the loop via one `uv run prevue review` (SECR-01, thin-workflow/fat-CLI).
- [ ] A real PR event end-to-end produces a sticky comment, then updates it in place on a second push (OUTP-01, DIFF-01, ENGN-02 — the live proof).
- [ ] The README documents the trigger matrix and the token recipe so adopters can reproduce the run.

This is the capstone that proves the skeleton works on a real runner.

Purpose: Everything until now is unit-tested in isolation; this plan exercises the real GitHub Actions + Copilot CLI + PR-comment path and locks the trust posture in YAML and docs.
Output: `.github/workflows/review.yml`, `tests/test_workflow_yaml.py`, `README.md`, and a confirmed live E2E run.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub PR event → privileged job | secrets + write token become available here |
| workflow YAML | the only place secrets enter and permissions are scoped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02 | Elevation of Privilege | trigger model | mitigate | `on: pull_request` ONLY — never `pull_request_target`; no PR-head checkout in the privileged job; statically asserted in `test_workflow_yaml.py` + zizmor (Pitfall 1) |
| T-04-PERM | Information Disclosure | over-broad token scope | mitigate | `permissions: { contents: read, pull-requests: write }` only — least privilege (SECR-01) |
| T-03 | Spoofing | token shadowing | mitigate | Explicit `COPILOT_GITHUB_TOKEN` + `GITHUB_TOKEN: github.token` as separate step env; Copilot never grabs the Actions token (Pitfall 7) |
| T-04 | Denial of Service (UX) | fork PRs | mitigate | Fork unsupported documented in README + early exit (Plan 06); empty-secrets 403 avoided (Pitfall 2) |
| T-01-SC | Tampering | action/CLI supply chain | mitigate | SHA-pin all actions; pin `@github/copilot@1.0.60`; `uv sync --locked` |
</threat_model>
