## Phase Goal

**As a** developer who opens a pull request, **I want to** automatically receive an AI-generated review summary comment, **so that** I get feedback without anyone running a review manually.

## Acceptance Criteria (this slice)
- [ ] One command (`prevue review`) drives the whole end-to-end loop (the walking skeleton's spine).
- [ ] Fork PRs are detected at the start and exit with a documented unsupported message (SECR-01/Pitfall 2).
- [ ] On any engine failure the run fails closed (non-zero exit) and the sticky comment is never touched (D-09).
