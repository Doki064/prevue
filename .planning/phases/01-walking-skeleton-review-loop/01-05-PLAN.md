## Phase Goal

**As a** developer who opens a pull request, **I want to** automatically receive an AI-generated review summary comment, **so that** I get feedback without anyone running a review manually.

## Acceptance Criteria (this slice)
- [ ] A pluggable `EngineAdapter` (ENGN-01) is implemented by a headless `CopilotCliAdapter` (ENGN-02).
- [ ] The diff is passed inline, fenced as data, with the agent granted ZERO tools (strongest injection posture).
- [ ] Auth guard + all failure modes (timeout/non-zero/empty) fail closed before any GitHub write (D-09/D-10).

<objective>
Implement the engine seam: the `EngineAdapter` ABC (locked ENGN-01 shape) and the concrete `CopilotCliAdapter` that shells out to the Copilot CLI headless, builds a fenced zero-tool prompt from the diff (excluding PR title/body), and maps every failure to a typed exception that fails the run. Design informed by the Plan 01 spike findings (A1 zero-tool viability, A2 PAT prefix, A3 timing).

Purpose: This is the only component that knows an AI vendor exists; it returns a `ReviewResult` and never posts to GitHub. It is the pluggable seam for future adapters.
Output: `engines/base.py`, `engines/copilot_cli.py` (+ `CopilotAuthError`, `EngineFailure`), `tests/test_copilot_adapter.py`.
</objective>
