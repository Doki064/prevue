# Quick Task 260701-ju8: Fix Phase 10 Thermos review findings (P0/P1/P2)

## Context

Phase 10 (boundary-contracts) thermo review found 12 actionable findings across
branch-audit (behavior/contract gaps, T-01..T-04) and code-quality (structural
debt, T-05..T-12) passes. 807 existing tests are green and must stay green.
This task fixes all 12 findings; the P3/info section (T-13..T-18) is explicit
follow-up debt and must NOT be touched.

Two items are internal refactors with no behavior change allowed:
- T-05: split `review.py` (1354 lines) into `review_config.py` / `review_classify.py`
  / `review_publish.py`, with `run_review()` shrunk to an orchestration pipeline.
- T-06: extract `_run_invocation()` / `_merge_retry_tokens()` out of
  `review_with_retry()` in `flow.py` (currently ~185 lines of retry/token spaghetti).

Order matters: fix small P0/P1 correctness items FIRST (tasks 1-6), THEN do the
two structural refactors (tasks 7-8) once the surrounding code is stable, so the
refactors move already-correct logic instead of having to re-verify twice.

One behavior change is intentional and required by T-03: `run_review()` currently
lets `UnknownEngineError` (bad `PREVUE_ENGINE` value) propagate uncaught. The fix
makes it a graceful skip like `NonFunctionalEngineError` already is. This means
the existing test `test_engine_selection_via_prevue_engine` in
`tests/test_review_flow.py` (which currently asserts `pytest.raises(UnknownEngineError)`)
must be rewritten to assert the new skip-check behavior — this is not a
"break a test" regression, it is the literal fix T-03 asks for.
